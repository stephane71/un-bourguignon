# Phase 2 — UI Design Contract

> Visual and interaction contract for the layout shell: sticky header with mobile burger menu, footer, and anchor scroll infrastructure. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage in Phase 2 |
|-------|-------|-------------------|
| xs | 4px | Icon internal gaps |
| sm | 8px | Nav link gap between text and underline, compact footer spacing on mobile |
| md | 16px | Header horizontal padding (mobile), footer vertical padding (mobile) |
| lg | 24px | Footer vertical padding (desktop), footer link gaps (desktop) |
| xl | 32px | Desktop nav link gaps (`gap-8`) |
| 2xl | 48px | Mobile menu link vertical gaps, minimum tap target size |
| 3xl | 64px | Mobile menu vertical centering area |

Exceptions:
- Header height: 64px mobile (`h-16`), 80px desktop (`h-20`) -- not a spacing token, a structural dimension
- `scroll-margin-top: 80px` (`5rem`) on `section[id]` -- matches header height + 16px breathing room
- Tap targets: 48px minimum on all interactive elements (`min-h-12 min-w-12`)

## Color

| Role | Value | Usage in Phase 2 |
|------|-------|-------------------|
| Dominant (60%) | `#FFFFFF` (white) | Header background, mobile menu overlay background, page background |
| Secondary (30%) | `#6B4C3B` (brun) | Footer background, body text color, nav link text, brand name text |
| Accent (10%) | `#6B4C3B` (brun) | CTA button fill, focus ring color |
| Text-on-dark | `#FFFFFF` (white) | Footer text and icons on brun background |
| Hover-on-dark | `#E8DCC8` (sable) | Footer link hover state on brun background |

Accent reserved for: header CTA button background, focus ring on all interactive elements (`focus:ring-brun`).

Color rationale:
- Header: white background, always opaque (locked decision from CONTEXT.md)
- Mobile menu overlay: white background -- consistent with header staying visible at top; ecru would create a disconnect
- Footer: brun (`#6B4C3B`) background with white text -- anchors the page bottom with warmth, creates clear visual bookend. White on brun contrast ratio: ~6.8:1, passes WCAG AA
- Social icons: white monochrome (`currentColor`) on brun footer -- clean, cohesive with earth-tone palette
- No destructive actions in this phase

## Component Inventory

### Header (`src/components/Header.tsx`)
- **Type:** Server Component (no `"use client"`)
- **Position:** `sticky top-0 z-50`
- **Background:** `bg-white` always opaque
- **Shadow:** None by default; `shadow-sm` added via JS when `scrollY > 0`
- **Height:** `h-16` mobile, `lg:h-20` desktop
- **Layout:** `flex items-center justify-between` inside Container
- **Brand:** `<a href="#">Stephane Maire</a>` in Lora serif, `text-xl font-bold text-brun`
- **Desktop nav:** `hidden lg:flex items-center gap-8`, 4 links + CTA button
- **Mobile nav:** Renders `<MobileMenu>` client component, `lg:hidden`

### MobileMenu (`src/components/MobileMenu.tsx`)
- **Type:** Client Component (`"use client"`)
- **Trigger:** Burger button, `min-h-12 min-w-12`, three horizontal lines icon
- **Overlay:** `fixed inset-0 top-16 z-40`, white background, `role="dialog"` `aria-modal="true"`
- **Links:** Centered vertically (`flex flex-col items-center justify-center h-full gap-8`), same 4 nav links + CTA
- **Open transition:** `motion-safe:transition-opacity duration-200` (fade in)
- **Close transition:** Same fade out, instant if `prefers-reduced-motion`
- **Burger/Close swap:** Burger icon when closed, X icon when open, same button position
- **ARIA:** `aria-expanded` on trigger, `aria-controls="mobile-menu"`, `aria-label` toggles between "Ouvrir le menu" / "Fermer le menu"
- **Focus management:** `inert` attribute on `<main>` when open; first link receives focus on open; focus returns to burger button on close
- **Dismiss:** Escape key closes menu; tapping any link closes menu and scrolls to section

### HeaderShell (optional consolidation)
- If scroll shadow logic is combined with MobileMenu into a single client wrapper, name it `HeaderShell.tsx`
- Manages: `isOpen` state for menu + `isScrolled` state for shadow
- Renders both the burger button and the scroll shadow class toggle

### Footer (`src/components/Footer.tsx`)
- **Type:** Server Component
- **Background:** `bg-brun text-white`
- **Padding:** `py-6`
- **Layout:** `flex flex-col sm:flex-row items-center justify-center gap-4 sm:gap-8`
- **Content order:** Social icons (LinkedIn, Facebook) | Phone | Email | Mentions legales
- **Link styling:** `min-h-12 inline-flex items-center` with `hover:text-sable transition-colors`
- **Social icons:** Wrapped in `<a>` with `min-h-12 min-w-12 inline-flex items-center justify-center` for 48px tap target
- **Phone:** `href="tel:+33662461643"`, display text "06 62 46 16 43"
- **Email:** `href="mailto:stephane-ei@un-bourguignon.com"`, display text "stephane-ei@un-bourguignon.com"
- **Mentions legales:** `href="/mentions-legales"`, display text "Mentions legales" (page created in Phase 4, 404 expected for now)

### Icon Components (`src/components/icons/`)
- **LinkedIn.tsx:** Inline SVG, 24x24, `fill="currentColor"`, `aria-hidden="true"`
- **Facebook.tsx:** Inline SVG, 24x24, `fill="currentColor"`, `aria-hidden="true"`
- **Burger.tsx:** Inline SVG, 24x24, three horizontal lines, `aria-hidden="true"`
- **Close.tsx:** Inline SVG, 24x24, X shape, `aria-hidden="true"`
- All icons accept `className?: string` and `size?: number` props

## Anchor Scroll Infrastructure

| CSS Rule | Value | Location |
|----------|-------|----------|
| `html { scroll-behavior: smooth; }` | Enables smooth anchor scrolling | `globals.css` |
| `@media (prefers-reduced-motion: reduce) { html { scroll-behavior: auto; } }` | Respects user motion preference | `globals.css` |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| A11Y-02: 48px tap targets | `min-h-12` on all nav links, burger button, footer links, CTA button |
| A11Y-03: Keyboard nav + focus | `focus:ring-2 focus:ring-brun focus:ring-offset-2` on all interactive elements; Tab cycles through all focusables; Escape closes mobile menu |
| WAI-ARIA Disclosure | Burger uses `aria-expanded`, `aria-controls`; overlay uses `role="dialog"`, `aria-modal="true"` |
| Focus trap | `inert` attribute on `<main>` when mobile menu is open |
| Focus restore | Focus returns to burger button when menu closes |
| Reduced motion | `motion-safe:` prefix on menu transitions; `scroll-behavior: auto` in `prefers-reduced-motion: reduce` |
