**src/app/layout.tsx** -- Replace the ENTIRE file with font loading setup. Write this exact content:

```typescript
import type { Metadata } from "next";
import { Lora, Instrument_Sans } from "next/font/google";
import "./globals.css";

const lora = Lora({
  subsets: ["latin"],
  display: "swap",
  variable: "--font-lora",
});

const instrumentSans = Instrument_Sans({
  subsets: ["latin"],
  display: "swap",
  variable: "--font-instrument",
});

export const metadata: Metadata = {
  title: "Un Bourguignon",
  description: "Site vitrine professionnel",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="fr" className={`${lora.variable} ${instrumentSans.variable}`}>
      <body className="font-sans">{children}</body>
    </html>
  );
}
```

Key points:
- `Instrument_Sans` uses underscore (Google Fonts multi-word convention for next/font)
- Both fonts use `subsets: ["latin"]` and `display: "swap"` to prevent FOUT
- `variable` option creates CSS custom properties `--font-lora` and `--font-instrument` on the html element
- These CSS variables are consumed by `@theme inline` in globals.css to create `font-serif` and `font-sans` Tailwind utilities
- `body className="font-sans"` sets Instrument Sans as the default body font
- `lang="fr"` is preserved (FOND-06)
- Both fonts are variable fonts (400-700 weight range), no `weight` parameter needed
