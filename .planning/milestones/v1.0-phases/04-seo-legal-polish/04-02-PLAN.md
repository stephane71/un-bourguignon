**1. Fix contrast issue in MethodeSection.tsx:**

Change line 57 from:
```
<span className="font-serif font-bold text-or-light text-4xl lg:text-5xl">
```
to:
```
<span className="font-serif font-bold text-argile text-4xl lg:text-5xl">
```

Reason: `text-or-light` (#D4A574) on white background has contrast ratio 2.23:1, which FAILS WCAG AA even for large text (requires 3:1). `text-argile` (#B8754C) on white has ratio 3.69:1, which PASSES large text AA (3:1). The step numbers at 36-48px bold qualify as large text.

Note: `marker:text-or` in ServicesSection.tsx list markers are DECORATIVE (non-text visual indicators) and exempt from WCAG contrast requirements. Do NOT change those.

**2. Add print stylesheet to globals.css:**

Append AFTER the existing `@media (prefers-reduced-motion)` block:

```css
@media print {
  header,
  footer,
  nav,
  .no-print {
    display: none !important;
  }

  body {
    color: #000 !important;
    background: #fff !important;
  }

  section {
    background: transparent !important;
    padding-top: 1rem !important;
    padding-bottom: 1rem !important;
    break-inside: avoid;
  }

  a[href^="tel:"]::after {
    content: " (" attr(href) ")";
    font-size: 0.875rem;
    color: #555;
  }

  #contact {
    border-top: 1px solid #ccc;
    padding-top: 1rem;
  }
}
```

This hides header/footer/nav during print, removes background colors for B&W optimization, reduces section padding for paper, preserves the contact section with a separator, and appends phone link URLs for print reference. CTA buttons are inside the header and sections that will print as plain text (anchor tags without button styling).

**3. Extend prefers-reduced-motion in globals.css:**

Replace the existing `@media (prefers-reduced-motion: reduce)` block with:

```css
@media (prefers-reduced-motion: reduce) {
  html {
    scroll-behavior: auto;
  }

  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This extends the existing scroll-behavior fix to also disable ALL CSS transitions and animations for users who prefer reduced motion.
